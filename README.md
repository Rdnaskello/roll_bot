# Roll Bot — инструкция для обычных людей

## Шуточная лицензия и пользовательское соглашение

Этот бот был написан для таких же ленивых жоп, как и автор, которым впадлу вручную очищать значки и по сто раз смотреть на одни и те же статы.

Используя этого бота, вы соглашаетесь с тем, что:

- это **сырая версия**;
- бот **может ошибаться**;
- бот **может неправильно распознать стат**;
- бот **может неправильно прочитать число**;
- бот **может нажать не туда**, если у вас неправильно настроены координаты;
- автор **не несёт ответственности** за ваши ресурсы, попытки, предметы, валюту, нервы, клавиатуру, мышку, монитор и моральное состояние;
- перед запуском на реальные ресурсы вы сами обязаны проверить работу бота через тест;
- если вы запустили бота, ушли пить чай, а он что-то сделал не так — это не магия, это надо было читать инструкцию.

На данный момент **нет полноценной поддержки очищения значков 8-го уровня**, потому что автор не подлый донатер и пока не имеет максимальных значков для нормального тестирования. Как только у автора появятся максимальные значки, эта часть будет доработана.

# 1. Что делает бот

Бот помогает автоматически очищать значки.

Он делает такой цикл:

1. нажимает кнопку **Очистить**;
2. смотрит, какой стат выпал;
3. смотрит, какое число выпало рядом со статом;
4. сравнивает это с выбранной целью;
5. если стат подходит — нажимает **Обмен**;
6. если стат не подходит — нажимает **Игнор**;
7. повторяет, пока не найдёт нужный результат или пока вы его не остановите.

---

# 2. Важное правило перед использованием

Бот не думает как человек. Он работает только по вашим настройкам.

Если неправильно настроить:

- координаты кнопок;
- область названия стата;
- область числа;
- шаблоны статов;
- цель поиска;

то бот может работать неправильно.

Перед запуском всегда делайте проверку через:

```text
6. Test recognition
```

# 3. Что должно быть в папке бота

После распаковки архива у вас должна быть папка примерно такого вида:

```text
roll_bot/
├── roll_bot_v5.py
├── start_bot.bat
├── config.json
├── targets.json
├── templates.json
├── requirements.txt
├── templates/
├── debug/
└── logs/
```

## Что за что отвечает

```text
roll_bot_v5.py      — сам бот
start_bot.bat       — запуск бота двойным кликом
config.json         — настройки координат, задержек и Tesseract
targets.json        — список целей, которые бот может искать
templates.json      — список сохранённых шаблонов статов
templates/          — картинки-шаблоны статов
debug/              — отладочные картинки
logs/               — логи, если они используются
requirements.txt    — список нужных Python-библиотек
```

## Что нельзя удалять

Не удаляйте:

```text
config.json
targets.json
templates.json
templates/
```

Если удалить `templates.json` или папку `templates/`, бот потеряет сохранённые шаблоны статов.

Папку `debug/` можно чистить, если она стала слишком большой. Она нужна только для проверки.

# 4. Установка: самый простой порядок

## Шаг 1. Распакуйте архив

Распакуйте архив в удобное место, например:

```text
Рабочий стол → roll_bot
```

Пример хорошего пути:

```text
C:\Users\Ваше_Имя\Desktop\roll_bot
```

Не запускайте бота из:

```text
C:\Windows\system32
```

---

## Шаг 2. Установите Python

Если Python уже установлен, этот шаг можно пропустить.

Если Python не установлен:

1. Откройте браузер.
2. Найдите в поиске:  
   ```text
   Python download
   ```
3. Скачайте Python с официального сайта.
4. Запустите установщик.
5. Очень важно: поставьте галочку:
   ```text
   Add Python to PATH
   ```
6. Нажмите Install.

После установки перезагрузите компьютер или хотя бы закройте и заново откройте консоль.

### Как проверить Python

Откройте папку с ботом.

В верхней строке папки, где написан путь, нажмите мышкой, напишите:

```text
cmd
```

и нажмите Enter.

Откроется чёрное окно.

Введите:

```bat
python --version
```

Если вы видите что-то вроде:

```text
Python 3.12.х
```

значит Python установлен.

Если пишет, что `python` не найден — переустановите Python и обязательно поставьте галочку `Add Python to PATH`.

---

## Шаг 3. Установите библиотеки для бота

Откройте папку с ботом.

В верхней строке папки напишите:

```text
cmd
```

и нажмите Enter.

В открывшемся окне введите:

```bat
python -m pip install -r requirements.txt
```

Дождитесь завершения установки.

Если файла `requirements.txt` нет, используйте команду:

```bat
python -m pip install opencv-python numpy pyautogui pytesseract keyboard
```

---

## Шаг 4. Установите Tesseract OCR

Tesseract нужен, чтобы бот мог читать числа.

Если Tesseract уже установлен — этот шаг можно пропустить.

Если не установлен:

1. Откройте браузер.
2. Найдите:
   ```text
   Tesseract OCR Windows install
   ```
3. Установите Tesseract.
4. Желательно установить его в стандартную папку:
   ```text
   C:\Program Files\Tesseract-OCR\
   ```

После установки проверьте, что файл существует:

```text
C:\Program Files\Tesseract-OCR\tesseract.exe
```

---

## Шаг 5. Проверьте путь к Tesseract

Откройте файл:

```text
config.json
```

через Блокнот.

Найдите строку:

```json
"tesseract_path": "C:\\Program Files\\Tesseract-OCR\\tesseract.exe"
```

Если Tesseract установлен в другом месте, укажите свой путь.

Обычно менять ничего не нужно.

# 6. Главное меню

После запуска появится меню:

```text
1. Start bot
2. Select target
3. Add target
4. Delete target
5. List targets
6. Test recognition
7. Calibrate coordinates and OCR regions
8. Show config
9. Exit
10. Save current stat as template
11. List templates
```

Что означает каждый пункт:

| Пункт | Что делает |
|---|---|
| `1. Start bot` | Запускает автоматическое очищение |
| `2. Select target` | Выбор цели, которую бот должен искать |
| `3. Add target` | Добавить новую цель |
| `4. Delete target` | Удалить цель |
| `5. List targets` | Показать все цели |
| `6. Test recognition` | Проверить, что бот видит на экране |
| `7. Calibrate coordinates and OCR regions` | Настройка кнопок и областей чтения |
| `8. Show config` | Показать настройки |
| `9. Exit` | Закрыть бота |
| `10. Save current stat as template` | Сохранить текущий стат как шаблон |
| `11. List templates` | Показать список шаблонов |

# 8. Как бот распознаёт статы

Бот не читает название стата как обычный текст.

Он сравнивает картинку текущего названия со своими сохранёнными шаблонами.

Пример:

Вы сохранили шаблон:

```text
Рейтинг Урона
```

В игре выпало:

```text
Рейтинг Урона 6.00%
```

Бот сравнивает картинку с шаблонами и пишет:

```text
Detected stat: Рейтинг Урона
```

А число справа он читает отдельно через OCR.

---

# 9. Что такое шаблон

Шаблон — это маленький скриншот названия стата.

Например:

```text
Рейтинг Урона
Рейтинг Уворота
Скорость
Ловкость
Улучшить вред Крита
```

Для каждого стата, который бот должен уметь распознавать, нужен шаблон.

---

# 10. Как создать шаблон стата

Допустим, в игре сейчас видно стат:

```text
Рейтинг Урона
```

В меню бота нажмите:

```text
10
```

Бот спросит название.

Введите:

```text
Рейтинг Урона
```

Нажмите Enter.

Бот сохранит шаблон.

Желательно сделать несколько шаблонов одного и того же стата:

```text
10 → Рейтинг Урона
10 → Рейтинг Урона
10 → Рейтинг Урона
```

Обычно хватает 2–5 шаблонов.

Проверить список шаблонов:

```text
11. List templates
```

---

# 11. Почему лучше иметь шаблоны похожих статов

Если бот знает только:

```text
Рейтинг Урона
```

а в игре выпал:

```text
Рейтинг Уворота
```

он может перепутать их, потому что эти названия похожи.

Поэтому для похожих статов лучше создать шаблоны всех вариантов:

```text
Рейтинг Урона
Рейтинг Уворота
Рейтинг Крита
Рейтинг Антикрита
Рейтинг Пробития
```

Идеальный вариант — создать шаблоны всех статов, которые могут выпадать.

Минимальный вариант — создать шаблоны:

1. статов, которые вы хотите ловить;
2. статов, которые похожи на нужные.

---

# 12. Что такое таргет

Таргет — это цель, которую бот должен найти.

Например:

```text
Скорость = 75000
Рейтинг Урона = 6.00%
Улучшить вред Крита >= 20
```

Таргет состоит из двух частей:

```text
название стата + нужное число
```

---

# 13. Как добавить таргет

В меню нажмите:

```text
3. Add target
```

Бот спросит:

```text
Stat text as shown in game:
```

Введите название стата точно так же, как в шаблоне.

Пример:

```text
Рейтинг Урона
```

Дальше бот спросит условие:

```text
1. exact value
2. >= value
3. > value
4. <= value
5. < value
6. from min to max
```

Что выбрать:

| Вариант | Значение |
|---|---|
| `1. exact value` | строго равно |
| `2. >= value` | больше или равно |
| `3. > value` | строго больше |
| `4. <= value` | меньше или равно |
| `5. < value` | строго меньше |
| `6. from min to max` | диапазон от и до |

Примеры:

Для строго `6.00%`:

```text
Choose condition: 1
Target value: 6
```

Для `6.00% или больше`:

```text
Choose condition: 2
Target value: 6
```

Для значения от `4` до `6`:

```text
Choose condition: 6
Min value: 4
Max value: 6
```

Потом бот спросит:

```text
Display name [...]
```

Это название для меню. Можно написать понятно:

```text
Рейтинг Урона 6%
```

Потом:

```text
Template threshold [0.72]:
```

Обычно просто нажмите Enter.

В меню нажмите:

```text
2. Select target
```

Бот покажет список целей.

Введите номер нужной цели и нажмите Enter.

# 15. Как проверить, что всё работает

Перед запуском обязательно используйте:

```text
6. Test recognition
```

Бот покажет примерно такое:

```text
Detected stat: Рейтинг Урона
Template score: 0.9981
Template file: templates/reyting_urona_...
OCR value raw: '6.00%'
Parsed value: 6.0

Target stat: Рейтинг Урона
Text OK: True
Value OK: True
Match result: True
```

Что означает:

| Поле | Значение |
|---|---|
| `Detected stat` | какой стат бот увидел |
| `Template score` | насколько уверенно распознал |
| `OCR value raw` | что OCR прочитал из числа |
| `Parsed value` | какое число получилось |
| `Text OK` | совпал ли стат |
| `Value OK` | совпало ли число |
| `Match result` | итоговая проверка |

Если видите:

```text
Text OK: True
Value OK: True
Match result: True
```

значит бот правильно нашёл выбранный таргет.

# 16. Как запустить очищение

Когда вы проверили таргет через `Test recognition`, можно запускать:

```text
1. Start bot
```

Бот начнёт очищать значки.

Сначала посмотрите на первые попытки. Не уходите сразу.

Если всё работает правильно, можно оставить бота работать, но всё равно помните: версия сырая, бот может ошибаться.

# 17. Калибровка

Калибровка нужна, если:

- бот кликает не туда;
- бот читает не тот стат;
- бот читает не то число;
- вы сдвинули окно игры;
- изменили масштаб;
- изменили разрешение экрана.

В меню выберите:

```text
7. Calibrate coordinates and OCR regions
```

Бот попросит указать:

1. кнопку **Очистить / Обмен**;
2. кнопку **Игнор / Вернуть**;
3. область названия стата;
4. область числа.

## Как указывать координаты

Бот будет писать:

```text
Move mouse to the required position and press Enter.
```

Это значит:

1. наведите мышку на нужное место;
2. не кликайте;
3. нажмите Enter в консоли.

---

# 18. Как правильно выделять область названия стата

Область названия должна содержать только название.

Хорошо:

```text
[ Рейтинг Урона ]
```

Плохо:

```text
[ иконка + Рейтинг Урона ]
[ Рейтинг Урона + 6.00% ]
[ половина названия ]
[ название + соседний текст ]
```

Если область слишком короткая, бот будет путать похожие статы.

---

# 19. Как правильно выделять область числа

Область числа должна содержать всё число полностью.

Хорошо:

```text
[ 6.00% ]
[ 15.00% ]
[ 75000.00 ]
```

Плохо:

```text
[ 6 ]
[ .00% ]
[ 15. ]
[ число + лишний соседний текст ]
```

Лучше захватить немного места слева и справа, чем обрезать число.

---

# 20. Если бот неправильно читает число

Иногда Tesseract может ошибаться.

Примеры:

```text
4.00%  → 400
10.00% → 1000
15.00% → 1500
```

Что сделать:

1. перекалибровать область числа;
2. выделить число шире;
3. захватить символ `%`;
4. сделать `Test recognition`;
5. посмотреть debug-картинки в папке `debug/`.

---

# 21. Если бот путает статы

Например, бот путает:

```text
Рейтинг Урона
Рейтинг Уворота
```

Что сделать:

1. создайте шаблоны для обоих статов;
2. сделайте 3–5 шаблонов каждого;
3. проверьте через `Test recognition`;
4. если всё ещё путает — увеличьте `Template threshold` при создании таргета.

Обычно можно поставить:

```text
0.80
```

или:

```text
0.85
```

Чем выше число, тем строже проверка.

# 27. Быстрый старт для тех, кому лень читать

1. Распакуй архив.
2. Установи Python.
3. Установи библиотеки.
4. Установи Tesseract.
5. Запусти `start_bot.bat`.
6. Выбери таргет через `2. Select target`.
7. Проверь через `6. Test recognition`.
8. Если всё правильно — `1. Start bot`.
9. `F8` — пауза.
10. `F9` или `ESC` — стоп.
11. Если бот ошибся — надо было читать инструкцию полностью.

---

# 28. Самый безопасный порядок перед тратой ресурсов

Перед тем как запускать бота на реальные ресурсы:

1. выберите нужный таргет;
2. сделайте `Test recognition`;
3. убедитесь, что `Detected stat` правильный;
4. убедитесь, что `Parsed value` правильный;
5. убедитесь, что `Text OK` и `Value OK` работают логично;
6. только потом запускайте `Start bot`;
7. первые несколько попыток смотрите за ботом.

# 29. Финальное предупреждение

Бот — это помощник, а не гарантия идеального результата.

Он может облегчить рутину, но не отменяет необходимость думать, проверять настройки и следить за первыми попытками.

Используйте аккуратно.
